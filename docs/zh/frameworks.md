# 引用的方法论

Skill 的工作流建立在多个成熟框架之上。写作 / 指导时**点名**框架，给本人一个可深入的抓手，也体现专业度。

## 简历写作

**Google XYZ 公式**（Laszlo Bock，前 Google SVP People Ops）

> "Accomplished [X] as measured by [Y], by doing [Z]."
> 做到了什么 / 如何度量 / 通过什么动作

每条要点回答三个问题：做了什么、影响如何度量、通过什么动作实现。这是提升简历质量的最大杠杆。

**一数字铁律**：每条简历要点必须含至少 1 个量化元素（人数、百分比、金额、时间、规模）。无数字的要点在复盘时删掉。

**CAR / PAR / STAR**（Challenge-Action-Result / Problem-Action-Result / Situation-Task-Action-Result）：短要点用 XYZ 太繁琐时的简化版。STAR 是主导形式，在 `profiles/stories.md` 里明确使用。

## 面试

**BEI — Behavioral Event Interviewing**（行为事件访谈，David McClelland，哈佛心理学家，方法论创始人）

大多数行为面试的基础。前提：过去的行为预测未来的行为。面试官会追问具体过往事件（"讲一次你……的时候"），不问假设。准备时用具体事件，不要泛泛总结。

**STAR**（Situation / Task / Action / Result）：BEI 问题的主导回答结构。`profiles/stories.md` 全部使用。

## 课题 / 基金申报

**Heilmeier Catechism** — 海尔美尔 9 问（George Heilmeier，前 DARPA 局长）

每份科研申报书都应回答的 9 个问题——即使申请模板没明确问，答出来更有说服力：

1. 你想做什么？（不用行话）
2. 现在是怎么做的？瓶颈在哪？
3. 你的新方法是什么？为什么会成功？
4. 谁关心？做成了意味着什么差别？
5. 风险与回报是什么？
6. 要花多少钱？
7. 要花多长时间？
8. 中期和终期如何衡量成功？
9. 退出策略是什么？

这是 `workflows/grant-application.md` 里"研究基础"段落的组织骨架。

## 职业定位

**T 型 / π 型 / M 型技能**（McKinsey / IDEO 推广）

单一垂直领域的深度 + 相邻领域的广度。资深岗位常需 π 型（两个深度）或 M 型（多个深度）。在阐述独特价值时使用。

**career capital 职业资本**（Cal Newport，*So Good They Can't Ignore You*）

稀有且有价值的技能构成杠杆。做 gap 分析时，问题不是"我缺什么"，而是"我有什么资本？哪些值得继续投资？"

## 求职策略

**Stretch target heuristic 求职拉伸启发式**（"1-2 级 + 1.5-3x 薪 + 70% 匹配"）

外跳的标准启发式：

- **职级**：瞄准比当前**高 1-2 级**的岗位。同级平跳不值得换工作成本；3 级以上通常不切实际（除非创业）
- **薪酬**：外跳**至少 1.2-1.5x** 当前总包；同时发生"公司类型跃迁"（成熟大厂 → 高速成长 startup）时**可拉升到 2-3x**
- **匹配门槛**：**~70% 匹配**即投，不等 100%。高阶岗雇主期待候选人入职后成长进入角色
- **渠道权重**：**warm intro（内推） >> cold apply（冷投）**（转化率差约 10 倍）。投递前先激活网络

**例外**：纯粹职业转型（降薪正常）、主动降速、已到市场天花板、早期职业（倍数更小）。这是 `workflows/jd-sourcing.md` 的底层逻辑。

## 尽调 / 核查

**Triangulation 三角定位**

每一条承重声明都需要 ≥2 个独立公开来源。定性研究的标准做法，在这里用于投递前的简历 / 申报书核查。`workflows/verification.md` 的基石。

## 目标设定

**SMART**（Specific / Measurable / Achievable / Relevant / Time-bound）

`todo.md` 里让事项具体化——含糊的条目会被改写成 SMART 形式。
