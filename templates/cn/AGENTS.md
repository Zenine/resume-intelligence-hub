## 简历分两大类

本项目服务于两类完全不同的简历场景：

| 类型 | 用途 | 数据源 | 侧重点 |
|------|------|--------|--------|
| **招聘应聘** | 求职投递 | `profiles/master.md` | 工作经历、项目成果、能力匹配、量化业绩 |
| **课题 / 基金申报** | 科研项目申报 | `profiles/research.md` | 论文、专利、标准、课题、研究方向、学术贡献 |

两类简历格式、语言风格、内容侧重完全不同，生成时必须区分。

## 工作流

### 1. JD 定制简历生成

当用户提供一个 JD / 岗位描述时：

1. **解析 JD**：提取关键信息（职位、单位、必需要求、加分要求、职责、资历）
2. **匹配经历**：从 master.md 中找出最相关的经历、项目/作品、能力
3. **策略制定**：确定突出哪些优势、如何重新表述经历以匹配 JD 关键词
4. **生成简历**：
   - 动词开头 + 量化成果
   - 关键词与 JD 对齐（ATS 友好）
   - 控制在 1-2 页
5. **生成 Cover Letter / 自荐信**（如请求）
6. **保存**：创建 `jobs/applications/{单位}-{职位}-{date}/` 目录

### 2. 课题 / 基金申报简历生成（仅科研轨启用时）

1. **确认基金类型**：不同基金格式差异较大，先问清楚
2. **读取 research.md**：完整科研成果
3. **按基金格式组织 + 按申报方向筛选**
4. **生成研究基础段落**：概述申请人在该方向的积累
5. **保存**：`科研/` 目录

### 3. 面试教练

- 基于 JD 和生成的简历，预判面试问题
- 为每个问题准备 STAR 格式的回答
- 专业 / 技术面试：梳理考点清单，推荐复习资料
- 行为面试：结合测评报告给出策略建议（如有）
- 可进行模拟面试（扮演面试官）

### 4. 职业咨询

- **能力差距分析**：目标职位要求 vs 现有能力
- **市场洞察**：行业趋势建议
- **路径规划**：从当前角色到目标角色
- **薪资谈判**：策略与话术

### 5. 投递前核查

高风险投递前必做：
- 列出简历/申报书里所有"承重"声明（学历、雇主、头衔、项目数据、客户名称、奖项等）
- 逐条交叉核查公开来源
- 产出 `verification/{date}-web-check.md`，标注 ✅ 已证实 / 🟡 待佐证 / 🔴 高风险
- 高风险项要么找到佐证，要么改写为更稳妥的措辞

## 输出原则

- 正文使用**中文**；专业术语可保留原文
- 简历输出语言按目标单位选择
- 所有建议基于真实经历，不捏造
- **量化优先 — 一条要点至少一个数字**（Google XYZ 公式：Accomplished [X] as measured by [Y], by doing [Z]）
- 面试故事按 **STAR** 结构（Situation / Task / Action / Result）
- 课题申报的研究基础叙事参考 **Heilmeier Catechism** 9 问
- 专业但有温度的语气

## 任务管理原则

- `todo.md` 只放**当前待办**，不留已完成项
- 完成的事项移到 `changelog.md`，按日期倒序
- 不要在 todo 里打 `[x]`；直接告诉 AI 归档

## 文件命名规范

- 投递目录：`{单位}-{职位}-{YYYY-MM-DD}/`
- 简历文件：`resume.md` 或 `resume-{language}.md`
- Cover Letter / 自荐信：`cover.md`
- 面试准备：`prep.md`
- 核查记录：`verification/{YYYY-MM-DD}-web-check.md`
