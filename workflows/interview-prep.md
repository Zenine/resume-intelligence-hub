# Workflow: Interview Preparation

## Input
- JD (or target company / role)
- Generated resume.md (if available)

## Steps

**Framework anchors**: behavioral interviews follow **BEI (Behavioral Event Interviewing, McClelland)** — past behavior predicts future behavior. Answers use **STAR (Situation / Task / Action / Result)**. Stories live in `profiles/stories.md`.

### 1. Predict questions
Generate three types based on the JD:

**Behavioral questions (BEI)**
- From responsibilities and soft skills, predict 8-12 common questions
- Senior roles likely include: worst decision, toughest organizational conflict, managing-up examples, handling layoffs / performance issues / restructuring
- Early-career: growth moments, collaboration examples, initiative stories

**Professional / craft questions**
- From hard requirements, list 10-20 core topics the interviewer may probe
- Tech management: system design, architecture trade-offs, tech-selection decisions, team productivity measures
- Individual contributors: depth questions + practical exercises
- Non-tech fields: scale to the field — case-study walkthroughs (consulting / law), portfolio critique (design), clinical reasoning (medicine), deal structuring (finance), curriculum design (education), etc.

**Questions for the interviewer** (the ones you ask)
- 3-5 questions that show depth of thought (avoid anything Googleable)

### 2. Prep STAR answers
Pull the best-matching stories from `profiles/stories.md`. If stories are thin:
- Surface the gap to the user immediately
- Don't invent stories on the fly

### 3. Topic review list
For the required competencies, list what the user should brush up on:
- Self-rate 🟢 / 🟡 / 🔵
- 🟡: spend time reviewing
- 🔵: prepare a "I haven't used it deeply, but the underlying concept is X, ramp-up path is Y" response

### 4. Behavioral strategy (if assessments available)
If `assessments/` has content (personality tests, 360 feedback, etc.):
- Read the latest (DISC / MBTI / Hogan / 16PF / 360 / etc.)
- Flag potential blind spots that may surface in interviews (over-caution, decisional hesitance, conflict avoidance, etc.)
- Prepare language to address these proactively

### 5. Write prep.md
Save to `jobs/applications/{company}-{role}-{date}/prep.md`:
- Categorized question list
- STAR answer outlines
- Topic review list
- Your questions for the interviewer
- Key risk points and response scripts

### 6. (Optional) Mock interview
On request, play the interviewer. Escalate difficulty each round, give feedback between rounds.
